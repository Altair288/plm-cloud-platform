# 🧭 一、项目总体目标

> **构建一个可动态定义分类、属性、枚举、编码规则的系统，供 PLM 建模与属性规范化使用。**

系统目标：

* 支持各类产品分类（开关按钮、电源类、连接器等）
* 每个分类可定义若干属性
* 属性支持不同类型（数值 / 文本 / 枚举 / 布尔 / 日期）
* 支持属性的枚举值定义（LOV）
* 所有分类、属性、枚举均通过**统一的编码规则自动生成唯一编码**
* 支持动态扩展（未来可对接 PLM 或 ERP）

---

# 🧩 二、功能模块清单

## 1️⃣ 分类管理（Category）

**功能目标**：定义产品大类，作为属性的上层容器。

| 功能点      | 说明                  | 备注                          |
| -------- | ------------------- | --------------------------- |
| 新增分类     | 用户输入分类名（例：开关按钮）     | 自动生成分类编码（CAT_BTN_001）       |
| 编辑分类     | 修改名称、描述、规则模板等       | 保留编码不变                      |
| 删除分类     | 检查是否存在属性引用          | 若有子属性需限制删除                  |
| 分类编码规则定义 | 每个分类可定义自己的编码前缀和序列规则 | 支持 `${year}`、`${month}` 等变量 |
| 分类列表查询   | 支持分页与搜索             | -                           |

---

## 2️⃣ 属性管理（Attribute Definition）

**功能目标**：定义每个分类下的属性。

| 功能点    | 说明                                    | 备注                              |
| ------ | ------------------------------------- | ------------------------------- |
| 新增属性   | 在指定分类下创建属性                            | 通过分类规则生成编码，如 `CAT_BTN_ATT_0003` |
| 属性类型定义 | enum / number / text / boolean / date | -                               |
| 设置单位   | 仅 type=number 时启用                     | 例：V、A、mm                        |
| 枚举绑定   | 当 type=enum 时，可绑定一个 LOV 集合            | 例：lov_btn_type                  |
| 属性描述   | 用户可添加解释说明                             | -                               |
| 属性顺序   | 可定义显示顺序                               | -                               |
| 删除属性   | 若有枚举关联需限制删除                           | -                               |

---

## 3️⃣ 枚举值管理（LOV / Enum）

**功能目标**：统一管理属性的枚举值集合。

| 功能点    | 说明                       | 备注                                       |
| ------ | ------------------------ | ---------------------------------------- |
| 新增 LOV | 定义一个枚举集，如 `lov_btn_type` | 自动生成编码                                   |
| 新增枚举项  | 在 LOV 下新增值               | 自动继承 LOV 编码，如 `CAT_BTN_ATT_0001_VAL_001` |
| 编辑/删除  | 支持修改描述、顺序                | 删除需校验引用关系                                |
| 导入导出   | 支持从 Excel 导入枚举           | -                                        |

---

## 4️⃣ 编码规则管理（Code Rule）

**功能目标**：为系统各对象提供编码策略。

| 功能点    | 说明                                | 备注                   |
| ------ | --------------------------------- | -------------------- |
| 规则作用对象 | 分类 / 属性 / 枚举 / 零部件等               | 例：attribute          |
| 可继承层级  | inheritFrom: category / attribute | 支持层级拼接               |
| 模板表达式  | pattern: `${categoryCode}ATT_`    | 支持变量替换               |
| 序列参数   | currentSeq / step / length        | 控制流水号                |
| 自动编号生成 | 调用接口返回完整编码                        | 例：`CAT_BTN_ATT_0001` |
| 规则冲突检测 | 防止重复前缀 / 规则循环继承                   | -                    |

---

## 5️⃣ 元数据管理（Metadata）

**功能目标**：定义系统的可配置性，让前后端动态生成表单或校验。

| 元数据类型  | 内容                                            |
| ------ | --------------------------------------------- |
| 属性类型定义 | enum, number, text, boolean, date             |
| 单位列表   | 常用单位表（V, A, mm, g 等）                          |
| 编码模板变量 | year, month, categoryCode, attributeCode, seq |
| 表单约束   | name 不重复，编码前缀唯一等                              |

# ⚙️ 三、系统接口（REST API 级）

| 模块   | 方法    | 路径                               | 功能说明                         |
| ---- | ----- | -------------------------------- | ---------------------------- |
| 分类   | POST  | `/api/categories`                | 新增分类（自动生成编码）                 |
| 分类   | GET   | `/api/categories`                | 查询所有分类                       |
| 分类   | PATCH | `/api/categories/:id`            | 修改分类信息                       |
| 属性   | POST  | `/api/categories/:id/attributes` | 新增属性（自动绑定编码）                 |
| 属性   | GET   | `/api/categories/:id/attributes` | 获取分类下属性列表                    |
| 枚举   | POST  | `/api/lov`                       | 新增枚举集合                       |
| 枚举   | POST  | `/api/lov/:id/values`            | 新增枚举值                        |
| 编码规则 | GET   | `/api/code-rule/:target`         | 获取指定类型的编码规则                  |
| 编码规则 | POST  | `/api/code-rule/generate`        | 生成编码（输入 target + categoryId） |
| 元数据  | GET   | `/api/metadata`                  | 获取系统元数据定义                    |

# 🧠 五、编码规则自动生成逻辑（核心业务规则）

1. 用户新增分类：

   * 读取 `code_rule[target='category']`
   * 根据规则生成分类编码
   * 写入 `category.code`
2. 用户新增属性：

   * 找到对应分类；
   * 读取 `code_rule[target='attribute']`；
   * 用 `pattern.replace("${categoryCode}", category.code)`；
   * 按序号生成编码；
3. 用户新增枚举：

   * 找到属性；
   * 读取 `code_rule[target='lov_value']`；
   * 替换 `${attributeCode}`；
   * 生成编码；
4. 每次生成编码时，`code_rule.currentSeq++`。

# 🧮 六、异常与校验逻辑

| 场景   | 校验内容              |
| ---- | ----------------- |
| 新增分类 | 检查名称重复            |
| 新增属性 | 同分类下名称唯一          |
| 新增枚举 | 同 LOV 下 value 不重复 |
| 删除分类 | 若存在属性则禁止删除        |
| 删除属性 | 若有枚举引用禁止删除        |
| 编码生成 | 当前规则是否存在，序列号是否超限  |
